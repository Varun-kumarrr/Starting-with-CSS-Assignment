Bem

Description & Rules for Bem are as follow:-

The Block, Element, Modifier methodology (commonly referred to as BEM) is a popular naming convention for classes in HTML and CSS. Developed by the team at Yandex, its goal is to help developers better understand the relationship between the HTML and CSS in a given project.

The class name for a block is all lowercase and hyphen-separated (e.g., button).

The class name for an element is separated from the block name by two underscores (e.g., button__text).

The class name for a modifier is separated from the block or element name by two hyphens (e.g., button--disabled or button__text--bold).

Use meaningful and descriptive names for blocks, elements, and modifiers.
Avoid using IDs for styling.

Blocks should be independent and not rely on the structure of other blocks.

Avoid deep nesting of selectors to prevent specificity issues. Each selector should be as low in specificity as possible.
